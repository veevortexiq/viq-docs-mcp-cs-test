    Categories

    Parent Category

For each category assigned to the product: - Check if parent category exists (if any) - Recursively create parent categories first - Match or create the category itself 2. Product is assigned to all destination category IDs 3. Category metafields are migrated for each created category

    Complex Rules

    Complex rules apply conditional logic based on modifier selections to adjust pricing, weight, or purchasing behavior.

    When Complex Rules Are Used

    Complex rules are triggered when a customer selects specific combinations of modifier values. For example: - "If customer selects 'Gift Wrapping' AND 'Express Shipping', add $15" - "If 'Large Size' is selected, increase weight by 2 lbs" - "If 'Discontinued Color' is selected, disable purchasing"

    Weight Adjusters

    Source Field

    Destination

    Migration Behavior

    Adjuster Type

    Adjuster Type

    percent, relative, absolute

    Adjuster Value

    Adjuster Value

    Weight adjustment value

    Complex Rules Migration Behavior

    1. ID Mapping: Modifier IDs and modifier value IDs are automatically mapped to destination IDs

    2. Deduplication: Existing matching rules in destination are preserved (not duplicated)

    3. Cleanup: Auto-generated single-condition rules are cleaned up to avoid conflicts

    4. Variant Mapping: If rules reference variants, variant IDs are mapped correctly

    Fields NOT Migrated (Auto-Generated): - Rule ID (new ID assigned) - Product ID (linked to new product) - Condition IDs (new IDs assigned)

    Example Complex Rule:

    Rule: "Premium Gift Package"
Conditions:
  - Modifier "Gift Wrapping" = "Deluxe Box" AND
  - Modifier "Gift Card" = "Yes"
Actions:
  - Add $25.00 to price (relative adjuster)
  - Add 0.5 lbs to weight
